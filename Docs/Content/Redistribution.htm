Redistribution

So you've written this bang up game and now you need to send it to someone and you're asking "How do I do this?" Well, an installer program like Inno Setup will help greatly. But what you'll really need is to know what files to include with your game or application. And without further ado:

Prerequisite packages for distribution:

These packages are installers that deliver required files for use by Gorgon and should be included with and executed by your installer package (at the very least your victims need to be informed that they need to install these items before running your application).

  • The SlimDX runtime.
  • The latest and greatest version of the DirectX runtime - The required files should be installed by SlimDX, but if not, you can use this.
  • .NET 3.5 SP1
  • Windows XP or better.

Along with those packages you will need to be sure that your applications bundles the following with its own installer:

Assembly (DLL/Executable) files required by Gorgon applications:
  • Gorgon.dll
  • GorgonFramework.dll - That is, if you're using the framework.

Required if you're using the compressed file system (BZip 2 compressed file system):
  • GorgonBZip2FileSystem.dll
  • ICSharpCode.SharpZipLib.dll

Required if you're using the raw input interface plug-in:
  • GorgonInput.dll

Required if you're using the GSound library:
  • GSound.dll

Unlike previous versions, v1.1.3246.24958 and beyond, will not require the Visual C++ 2008 SP1 Runtime.

Furthermore, there is no need for an x64 and x86 version of Gorgon, by setting the project properties to AnyCPU the framework will automatically select the proper platform version of SlimDX (SlimDX, both x86 and x64 if applicable, should be installed in the GAC to facilitate this).

If you wish to provide a platform specific binary you can specify x64 or x86 in your project properties and the runtime should pick the correct platform version (that is, x86 if you compile for x86 or x64 if you compile for x64).
