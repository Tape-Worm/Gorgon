Remarks

This takes an IntPtr representing a pointer to unmanaged memory, and aliases it to this object. This allows for safe use of the pointer when reading and writing the memory being pointed at.

When the GorgonPointerBase aliases an existing pointer, it merely provides access to the pointer. It is not responsible for the pointer, and thus will not free or allocate memory on its behalf. The responsibility of freeing memory assigned to a pointer is placed on the caller of this object.

Unlike the GorgonPointerAlias type, this type will derive the size of the data pointed at by pointer from the type indicated by T.

The type indicated by T is used to determine the amount of memory used by the array. This type is subject to the following constraints:

  • The type must be decorated with the StructLayoutAttribute.
  • The layout for the value type must be Sequential, or Explicit.

Failure to adhere to these criteria will result in undefined behavior. This must be done because the .NET memory management system may rearrange members of the type for optimal layout, and as such when reading/writing from the raw memory behind the type, the values may not be the expected places.

 Caution

Ensure that the size, in bytes, of the type indicated by T is not larger than the memory allocated. The object has no way to determine how much memory is allocated and therefore, if the size is too large, a buffer overrun may occur when reading or writing.

 Caution

This method does not support marshalled data (i.e. types with fields decorated with the MarshalAsAttribute). Usage of marshalled data with this type will give undefined results.
