Remarks

This takes a single value and pins it to a fixed location to allow reading/writing via a pointer. The size of the pointer will be dependant upon the size of the type indicated by T, in bytes.

When the GorgonPointer pins a value, it takes that value type and locks it down so the garbage collector cannot move the item around in memory. If unmanaged access to the value was taking place, and the value was not pinned, then the value could be freed while accessing it. By locking it down, we ensure that there's a fixed address for that data so that we can dereference it and access it via a pointer.

The type indicated by T is used to determine the amount of memory used by the array. This type is subject to the following constraints:

  • The type must be decorated with the StructLayoutAttribute.
  • The layout for the value type must be Sequential, or Explicit.

Failure to adhere to these criteria will result in undefined behavior. This must be done because the .NET memory management system may rearrange members of the type for optimal layout, and as such when reading/writing from the raw memory behind the type, the values may not be the expected places.

 Important

Creating a pointer with this method locks a value down so the garbage collector cannot finalize it or move it. This can lead to performance issues with garbage collection. The rule of thumb is to pin the value, quickly do the work required, and unpin it via the Dispose method on this object.

A call to Dispose is required when finished with a pointer created by this method. Failure to do so can lead to memory leaks until the object is finalized.

 Caution

This method does not support marshalled data (i.e. types with fields decorated with the MarshalAsAttribute). Usage of marshalled data with this type will give undefined results.
