Remarks

This wraps a pointer to unmanaged native memory that has been pinned to a fixed location. It provides better thread safety than the GorgonDataStream object and gives safe access when dereferencing the pointer to read or write data.

This pointer type takes a managed type, specified by the type parameter T, and pins its location in memory to a fixed address so a pointer can be used to access the underlying data. Because of this, performance issues for the garbage collector will arise if the pinned memory is held for too long. A value should only be pinned for the minimum amount of time needed and then released again.

Like the GorgonDataStream object, it provides several methods to read/write data to the memory pointed at by the internal pointer. With these methods the object can read and write primitive types, arrays, and generic value types.

The type indicated by T is used to determine the amount of memory, in bytes, that the type occupies. This type is subject to the following constraints:

  • The type must be decorated with the LayoutKind.
  • The layout for the value type must be Explicit, or MarshalAsAttribute.

Failure to adhere to these constraints will result in undefined behavior. This must be done because the .NET memory management system may rearrange members of the type for optimal layout, and as such when reading/writing from the raw memory behind the type, the values may not be the expected places.

 Important

Since this object pins a managed value to a fixed location in memory, it is required that users call the Dispose(Boolean) method as soon as the object no longer needs to be pinned. Failure to dispose this object will result in a memory leak since the garbage collector can no longer move or reclaim the the value that is pinned.

 Caution

None of the methods for this object allow for marshalling (i.e. members of a type with the MarshalAsAttribute). Passing data that requires marshalling, such as reference types, will not work and will give undefined results. Only primitive types, or value types that have value types (this rule still applies to the value type fields as well) or primitive types for its fields should be used.
