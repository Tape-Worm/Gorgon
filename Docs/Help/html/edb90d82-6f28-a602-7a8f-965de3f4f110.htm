Remarks

This wraps a pointer to unmanaged native memory that has been allocated by this object. It provides better thread safety than the GorgonDataStream object and gives safe access when dereferencing the pointer to read or write data.

When this object allocates memory, it will add pressure to the .NET garbage collector to ensure that it is aware that this object is consuming a much larger portion of memory than can be deduced by the garbage collector. This allows the GC to pick up the object for finalization when memory pressure is too high. The memory pressure is reset when this object is disposed or finalized.

Like the GorgonDataStream object, it provides several methods to read/write data to the memory pointed at by the internal pointer. With these methods the object can read and write primitive types, arrays, and generic value types.

When reading or writing generic value types, or arrays with value types as elements, it is important to ensure that the value type meets the following criteria:

  • The type must be decorated with the StructLayoutAttribute.
  • The layout for the value type must be Sequential, or Explicit.

Failure to adhere to these criteria will result in undefined behavior. This must be done because the .NET memory management system may rearrange members of the type for optimal layout, and as such when reading/writing from the raw memory behind the type, the values may not be the expected places.

 Important

This object allocates unmanaged memory. Since this memory is outside of the scope of the .NET garbage collector, it will be kept around until this object is finalized and could lead to memory leakage. Always call the Dispose(Boolean) method when an instance of this object is no longer needed.

 Caution

None of the methods for this object allow for marshalling (i.e. members of a type with the MarshalAsAttribute). Passing data that requires marshalling, such as reference types, will not work and will give undefined results. Only primitive types, or value types that have value types (this rule still applies to the value type fields as well) or primitive types for its fields should be used.
