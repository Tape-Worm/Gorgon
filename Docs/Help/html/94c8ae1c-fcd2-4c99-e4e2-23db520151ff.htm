Remarks

This constructor takes a value and pins it to a fixed location to allow reading/writing via a pointer. The size of the memory pointed at will be dependant upon the the size, in bytes, of the generic type parameter, T.

When the GorgonPointerPinnedT type pins a value, it takes that value type and locks it down so the garbage collector cannot move the item around in memory. If unmanaged access to the value was taking place, and the value was not pinned, then the value could be freed while accessing it. By locking it down, we ensure that there's a fixed address for that data so that we can dereference it and access it via a pointer.

The type indicated by T is used to determine the amount of memory used by the type. This type is subject to the following constraints:

  • The type must be decorated with the LayoutKind.
  • The layout for the value type must be Explicit, or MarshalAsAttribute.

Failure to adhere to these criteria will result in undefined behavior. This must be done because the .NET memory management system may rearrange members of the type for optimal layout, and as such when reading/writing from the raw memory behind the type, the values may not be the expected places.

 Important

A call to Dispose(Boolean) is required when finished with this pointer. Failure to do so can lead to memory leaks.
