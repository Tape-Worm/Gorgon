Remarks

This wraps a pointer to unmanaged native memory that has been allocated, pinned or aliased. It provides better thread safety than the GorgonDataStream object and gives safe access when dereferencing the pointer to read or write data.

There are several types that implement this interface, and each of them handle a pointer differently:

TypeDescription
GorgonPointer Allocates a block of unmanaged memory provides access to the pointer that points to the unmanaged block. This memory can be aligned and the memory allocated is owned by the object and must be freed manually.
GorgonPointerTypedT Allocates a block of unmanaged memory large enough to fit the type specified by the generic type parameter and provides access to the pointer that points to the unmanaged block. This memory can be aligned and the memory allocated is owned by the object and must be freed manually.
GorgonPointerPinnedT Pins a managed value type specified by the generic type parameter to a fixed location and provides raw access to the data for that type. This memory is aligned by the .NET memory management system. The handle returning the pointer to the memory is owned by the the object and must be freed manually.
GorgonPointerAlias Aliases an existing unsafe void *, or IntPtr (for languages that don't support unsafe access) and provides access to the memory pointed at by the pointer passed in. The memory is not owned by the object and will not be freed by this object.
GorgonPointerAliasTypedT Aliases an existing unsafe void *, or IntPtr (for languages that don't support unsafe access) and provides access to the memory pointed at by the pointer passed in. The type provided by the generic type parameter is used determine the size of the data for the type, in bytes. The memory is not owned by the object and will not be freed by this object.

Like the GorgonDataStream object, it provides several methods to read/write data to the memory pointed at by the internal pointer. With these methods the object can read and write primitive types, arrays, and generic value types.

Please visit the documentation for each of the GorgonPointer types in the see also section for more detailed information.
