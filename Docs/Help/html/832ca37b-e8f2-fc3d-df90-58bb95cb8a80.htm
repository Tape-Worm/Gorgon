Remarks

Upon creation, this object will set its default values to the properties that have the GorgonApplicationSettingAttribute. These defaults can easily be overridden in the object constructor. If the object contains any collection/array properties, those must be initialized in the constructor. These property types will not receive default values on startup.

When this object is created, the Path property is filled in with a path that uses the current users roaming directory, it is formatted like this:
Users\<Your username>\<ApplicationName>\<Name of your assembly>.config.xml
The <Name of your assembly> is the name of the assembly that your settings object is contained within.

If you wish to set the default property values after object construction, call the Reset method.

There may be a slight delay when constructing objects that inherit this type. This is because the expressions used to manipulate the object are being compiled and a fair bit of reflection is used to derive which properties are usable against those that are not. This means that the more members you have on your object, the more of a delay on creation.

Objects that inherit this class are only required to submit the applicationName parameter. This parameter just identifies which application is using the setting object type. The settingsVersion property can be set to null (Nothing in VB.Net) to disable version checking. Likewise, the log parameter may be set to null to disable logging for this object.
